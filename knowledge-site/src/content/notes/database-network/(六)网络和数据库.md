## 一、项目中用到哪些DB(数据库)？
### 1、介绍
常见的关系型数据库：MySQL、PostgreSQL、Oracle Database、Microsoft SQL Server

常见的非关系型数据库：MongoDB、Redis、Neo4j、Cassandra 内存数据库：Memcached、Redis

android开发时常引入Room依赖项，Room 是 Android 官方提供的一个持久化库，用于简化对 SQLite 数据库的操作。它是一个抽象层，封装了底层的 SQLite API，使得开发者可以通过面向对象的方式操作数据库，而无需直接编写复杂的 SQL 语句。Room 的目标是提供一个更简洁、更安全、更易于维护的数据库操作框架。

Room 数据库主要由以下三个核心组件组成：

### 2、Entity（实体类）
实体类用于定义数据库表的结构。每个实体类对应一个数据库表，类中的字段对应表中的列。

```java
@Entity(tableName = "user")
public class User {
    @PrimaryKey(autoGenerate = true)
    public int id;

    @ColumnInfo(name = "first_name")
    public String firstName;

    @ColumnInfo(name = "last_name")
    public String lastName;

    @ColumnInfo(name = "age")
    public int age;
}
```

+ `@Entity` 注解用于标记这是一个实体类。
+ `tableName` 属性用于指定数据库表的名称。
+ `@PrimaryKey` 注解用于标记主键字段。
+ `@ColumnInfo` 注解用于指定列的名称。

### 3、DAO（数据访问对象）
DAO 是一个接口，用于声明数据库操作方法。Room 会在编译时生成具体的实现类。DAO 方法可以使用注解来定义 SQL 操作。

```java
@Dao
public interface UserDao {
    @Insert
    void insert(User user);

    @Query("SELECT * FROM user WHERE first_name = :firstName")
    List<User> findUsersByFirstName(String firstName);

    @Update
    void update(User user);

    @Delete
    void delete(User user);
}
```

+ `@Dao` 注解用于标记这是一个 DAO 接口。
+ `@Insert` 注解用于定义插入操作。
+ `@Query` 注解用于定义自定义的 SQL 查询。
+ `@Update` 注解用于定义更新操作。
+ `@Delete` 注解用于定义删除操作。

### 4、RoomDatabase（数据库类）
数据库类是 Room 数据库的入口，用于管理数据库的创建和版本控制。它是一个抽象类，继承自 `RoomDatabase`。

```java
@Database(entities = {User.class}, version = 1)
public abstract class AppDatabase extends RoomDatabase {
    public abstract UserDao userDao();
}
```

+ `@Database` 注解用于标记这是一个数据库类。
+ `entities` 属性用于指定数据库中包含的实体类。
+ `version` 属性用于指定数据库的版本号。
+ 数据库类中必须包含一个抽象方法，返回对应的 DAO 接口。

### 5、Room 数据库的创建
创建 Room 数据库实例通常使用 `Room.databaseBuilder()` 或 `Room.inMemoryDatabaseBuilder()` 方法。

```java
AppDatabase db = Room.databaseBuilder(context, AppDatabase.class, "app_database")
                      .fallbackToDestructiveMigration()
                      .build();
```

+ `Room.databaseBuilder()` 方法用于创建一个持久化的数据库实例。
+ `fallbackToDestructiveMigration()` 方法用于在数据库版本升级时销毁旧表并重新创建（仅用于开发阶段，生产环境中应使用更安全的迁移策略）。

### 6、Room 的主要功能
1. 数据库操作
    - Room 提供了插入、查询、更新和删除等基本操作。
    - 支持批量操作，例如批量插入、批量删除等。
2. 事务支持
    - Room 支持事务操作，确保数据库操作的原子性。
    - 可以通过在 DAO 方法上添加 `@Transaction` 注解来声明事务。
3. 编译时检查
    - Room 在编译时会对 SQL 语句和数据库操作进行检查，确保 SQL 语句的正确性。
    - 如果存在错误，编译时会报错，而不是在运行时才发现问题。
4. 与 LiveData 和 Flow 集成
    - Room 支持与 Android 的 LiveData 和 Kotlin 的 Flow 集成，可以方便地实现数据的实时更新和响应式编程。
    - 当数据库中的数据发生变化时，绑定的 LiveData 或 Flow 会自动更新，从而通知 UI 层进行刷新。

### 7、Room 的版本迁移
当数据库结构发生变化时，需要进行版本迁移。Room 提供了多种迁移策略：

+ 自动迁移（Auto Migration）
    - 自动迁移适用于简单的结构变化，例如添加或删除列。
    - 需要在 `@Database` 注解中启用自动迁移功能。
+ 手动迁移（Manual Migration）
    - 手动迁移适用于复杂的结构变化，例如表的重命名、列的重命名等。
    - 需要通过 `Migration` 类编写迁移脚本。
+ 销毁并重建（Fallback to Destructive Migration）
    - 在开发阶段，可以使用 `fallbackToDestructiveMigration()` 方法销毁旧表并重新创建。
    - 但这种方法会丢失数据，因此不适用于生产环境。

### 8、Room 的优缺点
#### (1) 优点
1. 简化开发：通过注解和面向对象的方式，减少了直接编写 SQL 的工作量。
2. 编译时检查：减少运行时错误，提高开发效率。
3. 支持事务：确保数据操作的原子性。
4. 与 LiveData 和 Flow 集成：方便实现数据的实时更新。
5. 类型安全：通过注解和编译时检查，确保操作的类型安全。

#### (2) 缺点
1. 学习曲线：对于不熟悉关系型数据库和注解的开发者，可能需要一定时间来学习。
2. 性能开销：由于是抽象层，可能会比直接使用 SQLite 有一定的性能开销。
3. 灵活性限制：某些复杂的 SQL 操作可能不如直接使用 SQLite 灵活。

### 9、使用场景
Room 数据库适用于需要在本地存储数据的 Android 应用，例如：

+ 用户信息存储
+ 离线数据缓存
+ 应用设置存储
+ 数据持久化

## 六、Android中使用HTTP
Android 中包括两种方式来进行 HTTP 的请求：[HttpURLConnection](https://so.csdn.net/so/search?q=HttpURLConnection&spm=1001.2101.3001.7020) 和 HttpClient。

URL请求最常用的两种方式：GET请求与POST请求。

### 1、HttpURLConnection的使用
JDK的java.net包中已经提供了访问[HTTP协议](https://so.csdn.net/so/search?q=HTTP%E5%8D%8F%E8%AE%AE&spm=1001.2101.3001.7020)的基本功能的类：`HttpURLConnection`。`HttpURLConnection`是Java的标准类，它继承自`URLConnection`，可用于向指定网站发送GET请求、POST请求。

```java
package com.qf.demo;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.net.HttpURLConnection;
import java.net.URL;

/
 * GET请求示例
 *
 * @author 小明
 *
 */
public class GetDemo {

    public static void main(String[] args) {
        try {
            // 1. 得到访问地址的URL
            URL url = new URL(
                    "http://localhost:8080/Servlet/do_login.do?username=test&password=123456");
            // 2. 得到网络访问对象java.net.HttpURLConnection
            HttpURLConnection connection = (HttpURLConnection) url
                    .openConnection();
            /* 3. 设置请求参数（过期时间，输入、输出流、访问方式），以流的形式进行连接 */
            // 设置是否向HttpURLConnection输出
            connection.setDoOutput(false);
            // 设置是否从httpUrlConnection读入
            connection.setDoInput(true);
            // 设置请求方式
            connection.setRequestMethod("GET");
            // 设置是否使用缓存
            connection.setUseCaches(true);
            // 设置此 HttpURLConnection 实例是否应该自动执行 HTTP 重定向
            connection.setInstanceFollowRedirects(true);
            // 设置超时时间
            connection.setConnectTimeout(3000);
            // 连接
            connection.connect();
            // 4. 得到响应状态码的返回值 responseCode
            int code = connection.getResponseCode();
            // 5. 如果返回值正常，数据在网络中是以流的形式得到服务端返回的数据
            String msg = "";
            if (code == 200) { // 正常响应
                // 从流中读取响应信息
                BufferedReader reader = new BufferedReader(
                        new InputStreamReader(connection.getInputStream()));
                String line = null;

                while ((line = reader.readLine()) != null) { // 循环从流中读取
                    msg += line + "\n";
                }
                reader.close(); // 关闭流
            }
            // 6. 断开连接，释放资源
            connection.disconnect();

            // 显示响应结果
            System.out.println(msg);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

```java
package com.qf.demo;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStream;
import java.net.HttpURLConnection;
import java.net.MalformedURLException;
import java.net.URL;

/
 * POST请求示例
 *
 * @author 小明
 *
 */
public class PostDemo {

    public static void main(String[] args) {
        try {
            // 1. 获取访问地址URL
            URL url = new URL("http://localhost:8080/Servlet/do_login.do");
            // 2. 创建HttpURLConnection对象
            HttpURLConnection connection = (HttpURLConnection) url
                    .openConnection();
            /* 3. 设置请求参数等 */
            // 请求方式
            connection.setRequestMethod("POST");
            // 超时时间
            connection.setConnectTimeout(3000);
            // 设置是否输出
            connection.setDoOutput(true);
            // 设置是否读入
            connection.setDoInput(true);
            // 设置是否使用缓存
            connection.setUseCaches(false);
            // 设置此 HttpURLConnection 实例是否应该自动执行 HTTP 重定向
            connection.setInstanceFollowRedirects(true);
            // 设置使用标准编码格式编码参数的名-值对
            connection.setRequestProperty("Content-Type",
                    "application/x-www-form-urlencoded");
            // 连接
            connection.connect();
            /* 4. 处理输入输出 */
            // 写入参数到请求中
            String params = "username=test&password=123456";
            OutputStream out = connection.getOutputStream();
            out.write(params.getBytes());
            out.flush();
            out.close();
            // 从连接中读取响应信息
            String msg = "";
            int code = connection.getResponseCode();
            if (code == 200) {
                BufferedReader reader = new BufferedReader(
                        new InputStreamReader(connection.getInputStream()));
                String line;

                while ((line = reader.readLine()) != null) {
                    msg += line + "\n";
                }
                reader.close();
            }
            // 5. 断开连接
            connection.disconnect();

            // 处理结果
            System.out.println(msg);
        } catch (MalformedURLException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

```

### 2、HttpClient的使用
HttpClient相比较来说简化了HttpURLConnection对象对Session、Cookie的处理。简单来说，HttpClient就是一个增强版的HttpURLConnection，HttpURLConnection可以做的事情HttpClient全部可以做；HttpURLConnection没有提供的有些功能，HttpClient也提供了，但它只是关注于如何发送请求、接收响应，以及管理HTTP连接。

使用HttpClient发送请求、接收响应很简单，只要如下几步即可。

1、创建HttpClient对象。

2、如果需要发送GET请求，创建HttpGet对象；如果需要发送POST请求，创建HttpPost对象。

3、如果需要发送请求参数，可调用HttpGet、HttpPost共同的setParams(HttpParams params)方法来添加请求参数；对于HttpPost对象而言，也可调用setEntity(HttpEntity entity)方法来设置请求参数。

4、调用HttpClient对象的execute(HttpUriRequest request)发送请求，执行该方法返回一个HttpResponse。

5、调用HttpResponse的getAllHeaders()、getHeaders(String name)等方法可获取服务器的响应头；调用HttpResponse的getEntity()方法可获取HttpEntity对象，该对象包装了服务器的响应内容。程序可通过该对象获取服务器的响应内容。

```java
package com.qf.client;

import java.io.IOException;

import org.apache.http.HttpEntity;
import org.apache.http.client.ClientProtocolException;
import org.apache.http.client.methods.CloseableHttpResponse;
import org.apache.http.client.methods.HttpGet;
import org.apache.http.impl.client.CloseableHttpClient;
import org.apache.http.impl.client.HttpClientBuilder;
import org.apache.http.util.EntityUtils;

/
 * GET请求示例
 *
 * @author 小明
 *
 */
public class GetDemo {

    public static void main(String[] args) {
        // 1. 创建HttpClient对象
        CloseableHttpClient httpClient = HttpClientBuilder.create().build();
        // 2. 创建HttpGet对象
        HttpGet httpGet = new HttpGet(
                "http://localhost:8080/Servlet/do_login.do?username=test&password=123456");
        CloseableHttpResponse response = null;
        try {
            // 3. 执行GET请求
            response = httpClient.execute(httpGet);
            System.out.println(response.getStatusLine());
            // 4. 获取响应实体
            HttpEntity entity = response.getEntity();
            // 5. 处理响应实体
            if (entity != null) {
                System.out.println("长度：" + entity.getContentLength());
                System.out.println("内容：" + EntityUtils.toString(entity));
            }
        } catch (ClientProtocolException e) {
            e.printStackTrace();
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // 6. 释放资源
            try {
                response.close();
                httpClient.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

```

## 七、Http和Https区别
+ HTTP 是一个明文传输协议，不提供加密和安全保护，适合不需要高安全性的场景。
+ HTTPS 是 HTTP 的安全版本，通过 SSL/TLS 加密数据，提供数据加密、身份认证和完整性保护，适用于需要保护用户隐私和数据安全的场景。
+ 由于没有加密开销，HTTP 在某些情况下可能比 HTTPS 稍快。

## 八、TCP和UDP的区别
TCP（传输控制协议）和UDP（用户数据报协议）是两种常用的网络传输层协议

1. 连接方式

+ TCP：
    - 是一种面向连接的协议。在数据传输之前，必须先建立一个可靠的连接（通过三次握手）。
    - 适用于需要可靠传输的场景。
+ UDP：
    - 是一种无连接的协议。数据传输不需要预先建立连接，直接发送数据报。
    - 适用于对实时性要求较高、但对可靠性要求较低的场景。

2. 可靠性

+ TCP：
    - 提供可靠的数据传输，确保数据按顺序、完整地到达接收方。
    - 如果数据丢失或损坏，TCP 会自动重传数据。
    - 使用确认应答（ACK）和超时重传机制来保证数据的可靠性。
+ UDP：
    - 不保证数据的可靠传输，不提供确认应答机制。
    - 数据报可能丢失、重复或乱序到达，但不会自动重传。
    - 适用于对数据完整性要求不高，但对实时性要求较高的场景。

3. 数据传输方式

+ TCP：
    - 数据以字节流的形式传输，TCP 会将数据分割成多个数据段，并在接收端重新组装。
    - 适用于传输大量数据的场景。
+ UDP：
    - 数据以数据报的形式传输，每个数据报独立发送，大小通常有限制（一般不超过 64KB）。
    - 适用于传输小块数据的场景。

4. 速度

+ TCP：
    - 由于需要建立连接、确认应答和重传机制，TCP 的传输速度相对较慢。
    - 适用于对可靠性要求较高的场景。
+ UDP：
    - 由于无需建立连接和确认应答，UDP 的传输速度更快。
    - 适用于对实时性要求较高的场景，如视频流、在线游戏等。

5. 应用场景

+ TCP：
    - 适用于需要可靠传输的场景，例如：
        * 文件传输（FTP、SFTP）。
        * 网页浏览（HTTP/HTTPS）。
        * 电子邮件（SMTP、POP3）。
        * 远程登录（SSH）。
+ UDP：
    - 适用于对实时性要求较高、但对可靠性要求较低的场景，例如：
        * 视频流（RTP）。
        * 在线游戏。
        * 语音通话（VoIP）。
        * DNS 查询。

6. 头部开销

+ TCP：
    - TCP 头部长度为 20-60 字节（基本头部 20 字节，可选扩展头部）。
    - 提供更多的控制信息，如序列号、确认号、窗口大小等。
+ UDP：
    - UDP 头部长度固定为 8 字节。
    - 头部信息简单，仅包含源端口、目的端口、长度和校验和。

7. 流量控制和拥塞控制

+ TCP：
    - 提供流量控制（通过滑动窗口机制）和拥塞控制（通过慢启动、拥塞避免等机制）。
    - 适用于需要动态调整传输速率的场景。
+ UDP：
    - 不提供流量控制和拥塞控制。
    - 适用于对传输速率要求固定、且网络环境较好的场景。

8. 数据报的完整性

+ TCP：
    - 提供端到端的校验和，确保数据的完整性。
    - 如果数据报损坏，TCP 会要求重传。
+ UDP：
    - 提供简单的校验和，但不保证数据报的完整性。
    - 如果数据报损坏，UDP 不会自动重传。

总结

+ TCP 是一种面向连接、可靠的协议，适用于需要确保数据完整性和顺序的场景。
+ UDP 是一种无连接、不可靠的协议，适用于对实时性要求较高、但对可靠性要求较低的场景。

## 十一、HTTP2.0和HTTP3.0的区别
1. 协议基础

+ HTTP/2：
    - 基于 TCP 协议，使用二进制分帧技术，将 HTTP 数据分割成更小的帧，提高了传输效率。
+ HTTP/3：
    - 基于 QUIC 协议（Quick UDP Internet Connections），QUIC 是一种基于 UDP 的传输层协议，提供了更高效的传输机制。

2. 多路复用

+ HTTP/2：
    - 通过帧机制实现了多路复用，允许在同一个 TCP 连接上同时传输多个请求和响应，减少了连接数量，降低了延迟。
+ HTTP/3：
    - 基于 QUIC 的多路复用能力更强，支持无阻塞的多流并行传输。即使其中一个流遇到丢包或延迟，也不会影响其他流。

3. 队头阻塞

+ HTTP/2：
    - 由于依赖 TCP，仍然存在队头阻塞问题。一旦某个数据包丢失，整个连接上的数据流都会被阻塞。
+ HTTP/3：
    - 使用 QUIC 协议，彻底解决了队头阻塞问题。QUIC 支持多个独立的流，一个流的丢包不会影响其他流。

4. 连接建立和恢复

+ HTTP/2：
    - 需要 TCP 的三次握手和 TLS 握手，通常需要 1-2 个往返时间（RTT）。
+ HTTP/3：
    - QUIC 将 TLS 集成到传输层，支持 0-RTT 或 1-RTT 的连接建立，显著减少了连接建立时间。

5. 头部压缩

+ HTTP/2：
    - 使用 HPACK 算法压缩 HTTP 头部，减少了传输数据量。
+ HTTP/3：
    - 使用 QPACK 算法，为 QUIC 优化的头部压缩算法，进一步减少了头部传输的依赖和阻塞。

6. 连接迁移

+ HTTP/2：
    - 依赖 TCP，连接迁移能力有限，网络切换时需要重新建立连接。
+ HTTP/3：
    - 支持连接迁移，即使客户端的网络环境改变（如从 4G 切换到 WiFi），也能快速恢复连接。

7. 安全性

+ HTTP/2 和 HTTP/3：
    - 都要求默认使用 TLS 加密，确保数据传输的安全性。

8. 性能总结

+ HTTP/2：
    - 提高了传输效率，解决了 HTTP/1.1 的队头阻塞问题，但仍然依赖 TCP。
+ HTTP/3：
    - 通过 QUIC 协议，进一步优化了多路复用、连接建立和队头阻塞问题，提供了更低的延迟和更好的性能。

总结

HTTP/2 和 HTTP/3 都是 HTTP 协议的重大改进版本。HTTP/2 通过多路复用和头部压缩显著提升了性能，但仍然依赖 TCP 的可靠传输机制。HTTP/3 则通过 QUIC 协议，进一步解决了队头阻塞问题，提供了更快的连接建立和更高效的传输能力。
